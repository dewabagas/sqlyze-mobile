Definisi dan Fungsi Data Definition Language

Data Definition Language (DDL) adalah bagian penting dari SQL (Structured Query Language) yang digunakan untuk mendefinisikan dan mengelola struktur basis data, seperti tabel, indeks, dan tampilan (view). DDL memungkinkan pengguna untuk membuat, mengubah, dan menghapus struktur basis data sesuai dengan kebutuhan.

DDL berfokus pada bagaimana basis data terstruktur daripada bagaimana data disimpan atau diakses. Fungsi utama DDL meliputi:

  1. Membuat struktur basis data baru (tabel, indeks, dll.)
  2. Mengubah struktur basis data yang ada
  3. Menghapus struktur basis data yang tidak lagi diperlukan

Perbedaan DDL, DML dan DCL

Data Definition Language

DDL digunakan untuk mendefinisikan dan mengelola struktur basis data. Yang termasuk dalam DDL meliputi CREATE, ALTER, DROP, TRUNCATE, dan RENAME. Perintah ini memungkinkan pengguna untuk mendefinisikan struktur basis data (CREATE), mengubah struktur tersebut (ALTER), menghapusnya (DROP), menghapus semua data dalam struktur tersebut tanpa menghapus struktur itu sendiri (TRUNCATE), dan mengganti nama struktur tersebut (RENAME).

Data Manipulation Language

DML adalah bagian dari SQL yang digunakan untuk mengakses dan memanipulasi data. Perintah yang termasuk dalam DML adalah SELECT, INSERT, UPDATE, dan DELETE. SELECT digunakan untuk memilih data dari satu atau lebih tabel. INSERT digunakan untuk menambahkan baris baru ke tabel. UPDATE digunakan untuk memodifikasi baris yang ada di tabel. DELETE digunakan untuk menghapus baris dari tabel.

Data Control Language

DCL digunakan untuk mengontrol akses ke data dalam basis data. Perintah yang termasuk dalam DCL adalah GRANT dan REVOKE. GRANT digunakan untuk memberikan hak akses ke pengguna atau peran tertentu. Hak ini bisa mencakup kemampuan untuk menjalankan perintah DML dan/atau DDL, serta hak lain seperti hak untuk membuat prosedur yang disimpan dan fungsi. REVOKE digunakan untuk mencabut hak akses yang sebelumnya telah diberikan.
